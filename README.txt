# GRID portal v2

## Что внутри
- `uz/`, `ru/`, `en/` — готовые страницы по языкам
- `assets/data/projects.json` — проекты, мощность, дата запуска, галерея
- `assets/data/regions.json` — статистика по регионам
- `assets/data/team.json` — состав компании
- `assets/data/certificates.json` — сертификаты
- `assets/data/services.json` — услуги
- `assets/img/projects/` — фото по проектам
- `assets/img/team/` — фото сотрудников
- `assets/img/certificates/` — сканы сертификатов

## Как обновлять
1. Меняешь JSON в `assets/data/`
2. Кладёшь новые фото в соответствующие папки
3. При необходимости обновляешь ссылки на изображения в JSON
4. Загружаешь сайт как обычный статический архив

## Как считается большая цифра выработки
Для каждого проекта:
`мощность кВт × 4 кВт·ч в день × количество дней с даты запуска`
Потом всё суммируется автоматически на главной.

## Что стоит заменить перед публикацией
- реальные фотографии
- реальные сертификаты
- реальный состав компании
- фактические контакты и соцсети
- фактические данные по регионам и проектам

## Зачем так сделано
Сайт можно поддерживать как каталог:
данные отдельно, изображения отдельно, страницы почти не трогаются.
